# NOTES — 教学偏好与工作笔记

## 学习者画像
- 前端工程师，**后端零基础**；JS 扎实，熟悉 HTTP 调用、前端工程化。
- TypeScript：会简单 interface / enum，整体不扎实 → 需**由浅入深、就地带学**。
- 投入：每周 **20h+**，密集冲刺。
- 偏好：**中文**教学；**项目驱动 + 即时动手**；先讲够用的知识，立刻实践。

## 教学约定
- 课件用中文（技术术语保留英文原词 + 中文解释）。
- 每节课**聚焦一个点**，短小、可快速完成，带一个动手 / 自测环节（即时反馈）。
- 贯穿项目：**LinkPulse**（短链接 + 访问分析 + 订阅付费 SaaS）。
- TS 作为**副线**，跟随项目需要逐步加深（见 [[MISSION.md]]）。
- 严格引用**可信来源**（见 [[RESOURCES.md]]），不灌输未经核实的「脑内知识」。
- **课件形式（已定）**：零构建的自包含网页——共享 `lessons/lesson.css` + `lessons/lesson.js`，统一入口 `lessons/index.html`；课件仍可 `open` 直接看、零依赖。新增一课 = 写 `lessons/NNNN-*.html` + 在 `index.html` 加一张卡片。
- **未来项目**：后端有基础后，把「用 Next.js + TS 重做课程门户」作为一个正式实战项目（dogfooding + 作品集）。

## 教学校准（据用户反馈，2026-06-12）
- ⚠️ **核心教训**：用到的每个概念，必须**先讲清楚再用**；课件要自成一体，**绝不假设既有知识来填空**。（起因：0003 初版用了「路由 / 状态码 / 请求头」却没先教，被用户指出「课程很空」。）
- **知识起点**：前端已熟的 Web 概念（HTTP 方法、状态码、请求头等）→ 用到时**一两句快速复习 + 点明后端语境**即可，别花大篇幅；**后端新概念**（路由、流、中间件、认证…）→ 必须**完整讲透**。
- **课程颗粒度**：灵活按主题——复杂主题拆成多节、简单的合并。
- **结合实战（从 0004 起）**：在合适处补充「优秀用法 / 典型场景 / 常用方案 / 真实案例」，让用户边学边用。**按需融入、不强行堆砌**。（用户反馈，2026-06-12）
