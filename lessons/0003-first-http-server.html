Lesson 0003 · Node.js 后端之路

HTTP 基础，与你的
第一个服务器（原生 http 模块）

LinkPulse 的第一次心跳。 任何后端产品的起点，都是一个能接收请求、做出响应的服务器。 今天你写的这十几行，就是 LinkPulse 的骨架——后面的短链跳转、API、数据分析全长在它上面。 我们先不用框架，这样以后用 Express 时，你会清楚它到底替你做了什么。

1先翻转视角：你现在是「服务器」

HTTP 你在前端早就接触过——用 fetch 发请求、在 DevTools 的 Network 面板看请求和响应。现在视角要翻转：

以前你是客户端（发请求、收响应）；现在你写的是服务器（收请求、发响应）。 同一段 HTTP「对话」，你站到了另一头。

一次 HTTP「对话」由两部分组成，各自的「零件」如下（这些你在前端见过，这里快速过一遍，重点是它们在后端的含义）：

📨 请求 Request（你收到的）

  • 方法 method：想干什么。GET 取数据、POST 提交新数据、PUT 更新、DELETE 删除。
  • 路径 path：访问哪个地址，如 /、/api/health。
  • 请求头 headers：附加说明，如 Content-Type、Authorization。
  • 请求体 body：随请求带的数据（多见于 POST），可选。

📤 响应 Response（你发出的）

  • 状态码 status：结果如何。2xx 成功(200)、4xx 客户端错(404 找不到)、5xx 服务端错(500)。
  • 响应头 headers：如 Content-Type 告诉对方「我返回的是 JSON 还是 HTML」。
  • 响应体 body：真正返回的内容（文本、JSON、HTML…）。

🔑 关键转变：以前这些值是别人定的、你只负责读；现在作为服务器，是你来决定——返回什么状态码、什么响应头、什么内容。下面三节就是学怎么「读请求」和「定响应」。

2服务器骨架：createServer

Node 内置 http 模块，一个函数就能起服务器。最小可运行版本：

// 01-node-core/first-server.js
const http = require('node:http')

const server = http.createServer((req, res) => {
  res.statusCode = 200
  res.setHeader('Content-Type', 'text/plain; charset=utf-8')
  res.end('你好，这是我的第一个 Node 服务器！🎉')
})

server.listen(3000, () => {
  console.log('服务器已启动：http://localhost:3000')
})
  • http.createServer(handler) 创建服务器。每来一个请求，这个 handler 就被调用一次，所以它叫「请求处理器」1。
  • handler 拿到两个对象：req（你收到的请求）和 res（你要发出的响应）。
  • server.listen(3000) 让服务器开始监听 3000 端口。不调用它，服务器不工作1。
🔗 接上前两课：这个 (req, res) => {...} 回调，正是事件循环里被调用的回调。每来一个请求，事件循环就调用它一次——单线程、靠事件循环，就能源源不断地处理请求。地基用上了。

3读请求：req

req 是 http.IncomingMessage（本质是个可读流，下节接收 POST body 时会用到）。最常用三样，对应上面「请求」的零件：

req.method    // 方法："GET" / "POST" ...
req.url       // 路径，如 "/" 或 "/api/health"（不含域名和端口）
req.headers   // 请求头对象，如 req.headers['user-agent']

4定响应：res

res 是 http.ServerResponse，你用它「写回」。一次响应分三步：状态码 → 响应头 → 响应体：

res.writeHead(200, { 'Content-Type': 'application/json' })  // 状态码 + 响应头，一步到位
// 等价于：res.statusCode = 200; res.setHeader('Content-Type', 'application/json')

res.write('部分内容')      // 可选、可多次，往响应体里写
res.end('最后的内容')      // 结束响应并发送；⚠️ 每个请求只能调用一次

⚠️ res.end() 每个请求只能调用一次，表示「响应到此结束、发出去」。忘了调用，客户端会一直转圈等2。

5路由（Routing）：把不同请求分发到不同处理

什么是路由？ 一个服务器要处理很多种请求：访问首页、做健康检查、获取某条短链、创建新短链……它们显然不能都返回同一个东西。 路由（routing），就是根据请求的「方法 + 路径」，决定该运行哪段代码、返回什么响应。

打个比方 🏢：路由像公司前台接待——根据来访者要找谁（路径 /api/health）和要干什么（方法 GET/POST），把他们引到不同的房间办不同的事。

最朴素的路由，就是用 if / else 判断 req.url 和 req.method：

const http = require('node:http')

const server = http.createServer((req, res) => {
  console.log(`${req.method} ${req.url}`)               // 每个请求打印一行

  if (req.url === '/' && req.method === 'GET') {
    res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' })
    res.end('首页')
  } else if (req.url === '/api/health') {
    res.writeHead(200, { 'Content-Type': 'application/json; charset=utf-8' })
    res.end(JSON.stringify({ status: 'ok', uptime: process.uptime() }))
  } else {
    res.writeHead(404, { 'Content-Type': 'text/plain; charset=utf-8' })
    res.end('404 - 没有这个页面')
  }
})

server.listen(3000, () => console.log('http://localhost:3000'))

逐条读这个「前台」逻辑：

  • 方法是 GET、路径是 / → 返回纯文本「首页」，状态码 200。
  • 路径是 /api/health → 返回一段 JSON（注意响应头 Content-Type 改成了 application/json，告诉对方「我给的是 JSON」）。
  • 其它任何路径 → 返回 404，告诉客户端「没这个页面」。
💡 你可能已经想到：请求种类一多，这堆 if/else 会变得又长又乱。没错——这正是 Express 等框架要解决的问题（它让你写成 app.get('/api/health', …) 这样优雅地定义路由）。我们故意先用原生写法体会到痛点，等阶段 2 学框架时，你才会真切知道它好在哪。

📦 关于 require：这里用 require('node:http')（CommonJS），不用任何配置、node 文件名 直接能跑。它和你前端常用的 import（ESM）的区别，是下一课的主题，到时我们一起把项目正式搭起来。一次只学一个新东西。

⌨️ 动手：跑起你的第一个后端服务（10 分钟）

  1. 新建 01-node-core/first-server.js，抄入上面的「骨架版」代码（§2）。
  2. 终端运行 node 01-node-core/first-server.js → 看到「服务器已启动」。
  3. 浏览器打开 http://localhost:3000 → 看到问候语。🎉 你跑起了人生第一个后端服务！

  6. 看运行服务器的终端：每次访问都打印一行 GET /xxx —— 这就是事件循环在帮你调用回调。
  7. 按 Ctrl + C 停止服务器。
