# Mission: 从前端工程师成长为能独立交付产品的全栈开发者（Node.js 后端方向）

## Why
我是一名前端工程师，目标是**独立做出并上线能真实收费的 SaaS 产品**。要做到这一点，我必须补齐后端能力——从 Node.js 入手，打通网络、数据库、设计模式、部署运维的全链路，最终能**一个人把产品从代码写到上线运营**。

## Success looks like
- 能用 **Node.js + TypeScript** 从零设计并实现一个带认证、数据库、计费的 SaaS 后端
- 能独立设计合理的**数据库模型**，并用 SQL / Prisma 高效、安全地读写
- 能把应用**容器化（Docker）**、配置 **CI/CD**，并部署到**真实域名**上线运营
- 能讲清楚后端核心机制（事件循环、HTTP、认证、缓存、队列）而不只是「会用」
- TypeScript 从「会写 interface」进阶到能用**泛型、工具类型、装饰器**写出类型安全的后端

## Constraints
- 时间：每周 **20h+**，密集冲刺；目标约 **5–6 个月**达到能独立上线 SaaS MVP
- 起点：**后端零基础**；前端经验扎实（熟悉 JS、HTTP 调用、前端工程化）
- TypeScript 基础薄弱（会简单 interface、enum），需在过程中**由浅入深就地带学**
- 学习方式：**项目驱动 + 即时动手**；偏好**中文**教学
- 贯穿项目：**LinkPulse** —— 带数据分析与订阅付费的短链接 SaaS

## Out of scope（暂时不追，保护学习节奏）
- 前端框架本身（已掌握，后期才做整合）
- 算法刷题 / 八股面试（目标是做产品，不是求职）
- 其他后端语言（Go / Java / Python）——先把 Node 这条线走深
- Kubernetes / 大规模分布式架构（独立产品阶段用不到，远期再说）
